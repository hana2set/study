# 서브 테이블(1:N)의 데이터 존재 여부 조회
## 스칼라 서브쿼리 VS. LEFT JOIN (GROUP BY / DISTINCT)
### 가정 
- 조회 데이터 작음
- 서브 테이블 ID에 인덱스 걸려있음.

### 결론
- 손익분기점: 메인 테이블의 **조회 건수가 전체 데이터의 5~10%**

### 원인
- 스칼라 서브쿼리는 Nested Loop 방식.
  - Select문을 조회데이터 만큼 날림.
- LEFT JOIN은 인라인 뷰 방식
  - 서브 테이블에서 ID를 찾아야하기 때문에, 테이블 전체를 조회해야함.
  - Full Scan하여 GROUP 하는 방식
- 즉 **일반적인 조회에서는 스칼라 서브쿼리, 엑셀 다운 등 전체 조회에서는 LEFT JOIN**
